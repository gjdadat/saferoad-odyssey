# 개발 완료 및 기능 안내 (SafeRoad - 실시간 공공데이터 및 정적 호스팅 지원)

지도를 움직여 수동으로 찾는 불편함을 덜어드리기 위해 행정구역 선택 기능을 구현한 데 이어, **백엔드가 작동하지 않는 정적 호스팅(GitHub Pages) 환경에서도 실시간 공공데이터가 온전히 작동할 수 있도록 하이브리드 연동 로직** 개발이 완료되었습니다!

## 📁 파일 구조 및 목적
- `data.js`: 전국 주요 광역시/도 및 하위 지자체(구군)의 실제 중심부 위경도 좌표 매핑 테이블(`regionMapData`) 설계.
- `index.html`: 지도 뷰 영역 상단에 2단계 셀렉트 박스(시도 / 시군구), 조회 버튼 및 우측 상단 API 키 저장용 설정 모달 마크업.
- `style.css`: 반투명 어두운 네온 스타일의 지역 컨트롤러 컨테이너 및 셀렉트 박스, 모바일 전용 D-pad, API 설정 모달 스타일링.
- `app.js`: 백엔드 접속 실패 시 **브라우저 로컬 저장소(LocalStorage)** 및 **CORS 프록시(allorigins)**를 경유하여 공공데이터 포털 API(`https://apis.data.go.kr`)에 직접 조회하여 데이터를 뿌려주는 하이브리드 연동 알고리즘 추가.
- `server.js`: 로컬 구동 시 공공데이터 API 키를 `config.json`에 숨겨 중계해주는 Proxy 로직 제공.

## 🚀 깃허브 페이지(GitHub Pages) 실시간 데이터 사용 방법

백엔드가 없는 깃허브 웹서버 환경에서도 이제 실시간 데이터를 가져올 수 있습니다.

### 1. API 인증키 설정
1. 깃허브 웹 주소(`https://gjdadat.github.io/saferoad-odyssey/`)에 접속합니다.
2. 우측 상단의 **[⚙️ API 설정]** 버튼을 클릭합니다.
3. 정적 호스팅 감지 안내문구를 확인한 후, 공공데이터포털(data.go.kr)에서 발급받은 '도로교통공단 사고다발지정보' 일반 인증키(Decoding 키 권장)를 입력하고 **[저장하기]**를 누릅니다.
4. 화면이 새로고침된 후, 상단 배지가 **`실시간 공공데이터 연동 중 (웹)`** 상태로 변경됩니다!

### 2. 가상 체험 모드 복구
* 만약 입력한 키를 삭제하고 다시 가상 모드로 돌리고 싶다면, **[⚙️ API 설정]**을 누르고 입력창을 **완전히 비운 상태로 [저장하기]**를 누르시면 됩니다.

## 💻 로컬 PC 구동 방법 (백엔드 포함)

1. 터미널을 열고 프로젝트 폴더 경로(`C:\saferoad-odyssey`)로 이동합니다.
2. `npm install` 명령어로 필요한 패키지(express, cors 등)를 설치합니다.
3. `node server.js` 명령어로 로컬 서버를 가동합니다.
4. 브라우저로 `http://localhost:3000`에 접속합니다.
5. 로컬 PC 환경에서는 입력한 API 키가 브라우저가 아닌 서버의 `config.json`에 안전하게 숨겨져 저장됩니다.

## 🗺️ 주요 조작 흐름
* **지역 2단계 조회**: 드롭다운에서 시도와 구군을 선택하고 조회하면 지도가 그 구역으로 자동 비행(`flyTo`)합니다.
* **통계 및 게임 연동**: 지도 위의 핀을 탭하면, 해당 지역의 인명피해 비율 분석 차트가 로드됨과 동시에 그 지역의 보행 위험요소(무단횡단 차량, 신호등, 골목길 모퉁이 퀴즈 등)를 모의 체득할 수 있는 등교 미로찾기 게임이 즉시 생성됩니다.
